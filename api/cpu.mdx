The `CPU` struct is the heart of Neodymium VM, responsible for executing instructions, managing registers, and coordinating the RAM, Stack, and Screen components.

## Overview

The CPU contains 8 general-purpose registers, three status flags (zero, underflow, overflow), and orchestrates all VM operations through its instruction set.

- Initializes the RAM module
- Creates the Stack at address 0xCF00 in RAM
- Creates the Screen at address 0xA000 in RAM
- Allocates 8 general-purpose registers, all initialized to 0
- Sets all flags (zero, underflow, overflow) to false

## Public Methods

### tick()

```cpp
int tick()
```

Executes a single instruction cycle.

Defined in: `cpu.cpp:39`

<ResponseField name="return" type="int">
  Returns -1 if execution should continue, or an exit code (0-255) if a HALT instruction was executed.
</ResponseField>

**Behavior:**
- Fetches the next opcode from RAM at the current program counter (PC)
- Decodes and executes the instruction
- Updates registers, flags, and memory as needed
- Returns -1 for normal instructions or an exit code for HALT instructions (0xFF, 0xFE, 0xFD)

**Example:**

```cpp
CPU cpu;
// Load program into cpu.ram.memory...
cpu.ram.pc = 0x0000; // Set program counter to start

int result = cpu.tick(); // Execute one instruction
if (result == -1) {
    // Continue execution
} else {
    // Program halted with exit code
    std::cout << "Program exited with code: " << result << std::endl;
}
```

### run()

```cpp
int run()
```

Executes instructions continuously until a HALT instruction is encountered.

Defined in: `cpu.cpp:518`

<ResponseField name="return" type="int">
  Returns the exit code from the HALT instruction (0-255).
</ResponseField>

**Behavior:**
- Continuously calls `tick()` in a loop
- After each instruction, calls `screen.tick()` to update the display
- Continues until `tick()` returns a non-negative value (HALT)
- Returns the exit code from the HALT instruction

**Example:**

```cpp
CPU cpu;
// Load program into cpu.ram.memory...
cpu.ram.pc = 0x0000;

int exit_code = cpu.run(); // Run until HALT
std::cout << "Program completed with exit code: " << exit_code << std::endl;
```

</ParamField>

**Behavior:**
- Sets `overflow` to `true` if `num > 255`

- All flags are set to `false` if the corresponding condition is not met

## Instruction Set

The CPU supports a wide range of instructions organized into categories:

- **Data Movement** (0x00-0x04): MOV operations
- **Bitwise Logic** (0x10-0x12): NOT, AND operations
- **Control Flow** (0x20-0x2F): JMP, CMP, conditional jumps (JZ, JNZ, JU, JNU, JO, JNO)
- **Stack Operations** (0x30-0x32): PUSH, POP
- **Arithmetic** (0x40-0x4F): ADD, SUB, INC, DEC, MUL, DIV, PWR, SQRT, FSQRT, MOD
- **Subroutines** (0x50-0x52): CALL, RET
- **Memory Access** (0x60-0x63): STORE operations
- **Halt** (0xFD-0xFF): HALT with various exit code options

See the full instruction reference for detailed opcode documentation.

## Example Usage

```cpp
#include "cpu.h"

int main() {
    CPU cpu;
    
    // Load a simple program: MOV $0, #42; HALT
    cpu.ram.memory[0x0000] = 0x02; // MOV $x, #immediate
    cpu.ram.memory[0x0001] = 0x00; // Register 0
    cpu.ram.memory[0x0002] = 42;   // Immediate value
    cpu.ram.memory[0x0003] = 0xFF; // HALT
    
    // Run the program
    int exit_code = cpu.run();
    
    // Check register value
    // Note: registers is private, but you can access it through memory operations
    
    return exit_code;
}
```
