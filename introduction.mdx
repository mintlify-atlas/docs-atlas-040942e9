# Neodymium Virtual Machine

Neodymium VM is an open-source 8-bit virtual machine that simulates a complete computing environment including a CPU, RAM (up to 64KB), an in-RAM stack, and a virtual screen.

## What is Neodymium VM?

Neodymium VM provides a fully functional 8-bit computing environment for executing binary programs. It's designed for virtualization, learning low-level programming concepts, and experimentation with virtual machine architecture.

The VM executes binary programs from `.bin` files, processing instructions through its custom instruction set architecture with support for arithmetic operations, memory manipulation, control flow, and more.

## System Architecture

### CPU Registers

The CPU contains 8 general-purpose registers accessible via bytecodes:

- `$a` through `$h` (addresses `0x00` to `0x07`)
- Special "always zero" register at address `0xFF` for addressing operations

### CPU Flags

The CPU maintains three status flags updated by arithmetic and comparison operations:

- **Zero flag**: Set when the result equals zero
- **Overflow flag**: Set when a result exceeds 255
- **Underflow flag**: Set when a result is less than 0

### Memory Layout

```
0x0000 - 0x9FFF    Program code and general data
0xA000 - 0xCEFF    Virtual screen framebuffer (256 bytes)
0xCF00 - 0xCFFF    Stack memory (256 bytes)
```

<Note>
The stack grows from 0xCF00 to 0xCFFF and is managed automatically by PUSH, POP, CALL, and RET instructions.
</Note>

## Instruction Set

Neodymium VM supports a comprehensive instruction set including:

- **Data Movement**: MOV, PUSH, POP, STORE
- **Arithmetic**: ADD, SUB, MUL, DIV, INC, DEC, MOD, PWR, SQRT
- **Logic**: AND, NOT
- **Control Flow**: JMP, CALL, RET with conditional jumps (JZ, JNZ, JO, JNO, JU, JNU)
- **Comparison**: CMP
- **System**: HALT, NOP

All instructions operate on 8-bit values with support for both register and immediate operands.

## Use Cases

### Learning Low-Level Programming

Neodymium VM provides an accessible environment for understanding:
- CPU architecture and instruction execution
- Memory management and addressing modes
- Stack-based function calls
- Flag-based conditional logic

### Virtual Machine Experimentation

Experiment with:
- Binary program execution
- Custom instruction sets
- Memory-mapped I/O (screen)
- Stack operations

### Virtualization Projects

Build virtualized systems on top of Neodymium's foundation:
- Custom operating systems
- High-level language interpreters
- Embedded system simulations

<Warning>
Neodymium VM currently supports UNIX-based operating systems only (Linux, macOS). Windows is not supported.
</Warning>

## System Requirements

Minimal requirements for running Neodymium VM:

- **RAM**: 128 KB
- **CPU**: 1 core
- **Storage**: 8 MB
- **OS**: UNIX-based (Linux, macOS)
